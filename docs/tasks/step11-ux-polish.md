# Step 11: UX 改善 & 本番化

## 目的

実用レベルにするための改善を行う。

---

## 改善項目

### 1. スロットリング最適化

scene-delta の送信間隔を状況に応じて調整する。

- ドラッグ中: 50ms（20fps）
- ドラッグ終了時: 即座に最終値を送信
- アイドル時: 送信なし

### 2. 接続断リカバリ

- WebSocket 切断時に自動再接続（3秒間隔、既に Step 3 で実装済み）
- 再接続時に scene-request を送信して状態を再同期

### 3. pipe/index.html からのリンク

pipe の index.html に Scene Sync ビューアへのリンクを追加する。

    <a href="/pipe/scene.html">3D Scene Sync</a>

ルームコードが設定されている場合は room パラメータ付きでリンクする。

### 4. ルームコード UI

scene.html にルームコード入力・生成機能を追加する。
pipe/index.html と同じルームコード体系を使用する。

- URL に ?room= がなければ IP ベースの自動ルームを使用
- ルームコード入力欄を左上に小さく表示

### 5. モバイル対応

- タッチ操作での OrbitControls（ピンチズーム・2本指回転）
- ダブルタップで選択
- TransformControls のタッチ操作

### 6. エラーハンドリング

- glB ロード失敗時のフォールバック（Box 表示 + エラーバッジ）
- piping-server タイムアウト時のリトライ
- WebSocket メッセージパースエラーの安全な無視

### 7. パフォーマンス

- 大量オブジェクト時の scene-state サイズ制限
- scene-delta のバッチ送信（複数オブジェクト変更を1メッセージにまとめる）
- glB の Draco 圧縮（Three.js DRACOLoader 導入）
